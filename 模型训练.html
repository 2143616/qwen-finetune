1 项目概述与环境介绍

🎯 实验目标

使用 LLaMA-Factory 框架，在 NVIDIA DGX Spark 上通过 QLoRA (4-bit) 技术微调 Qwen2.5-7B-Instruct 模型，使其具备金融领域问答能力。

🧰 技术栈

🦙 LLaMA-Factory
统一微调框架，支持 100+ 模型，零代码 Web UI
🤖 Qwen2.5-7B
阿里通义千问 2.5 系列，7B 参数，中文优化
⚡ QLoRA (4-bit)
4-bit 量化 LoRA，大幅降低显存需求
🔲 DGX Spark
NVIDIA Grace Blackwell 紧凑型 AI 工作站，128GB 统一内存

📐 微调原理简述

QLoRA (Quantized Low-Rank Adaptation) 结合了两种关键技术：

  • 量化（Quantization）：将模型权重从 16-bit 压缩到 4-bit，显存占用降低约 75%
  • LoRA（Low-Rank Adaptation）：冻结原始权重，仅训练少量低秩适配矩阵（参数量仅为原始模型的 0.1%-1%）

通过 QLoRA，一块 8GB 显存的 GPU 即可微调 70 亿参数的大模型，而全参数微调需要约 120GB 显存。

💡

核心优势：QLoRA 在将显存需求降低 20 倍的同时，保留了全参数微调 95% 以上的效果，是消费级硬件微调大模型的首选方案。

4 环境搭建

请按照以下步骤在你的 DGX Spark 上搭建完整的微调环境。

步骤 1：安装 Miniconda（如未安装）

wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-aarch64.sh
bash Miniconda3-latest-Linux-aarch64.sh
# 按提示完成安装，重启终端或 source ~/.bashrc
⚠️

DGX Spark 是 ARM64 (aarch64) 架构，请务必下载 aarch64 版本的 Miniconda，不要使用 x86_64 版本！

步骤 2：创建 conda 环境

conda create -n llamafactory python=3.11 -y
conda activate llamafactory

步骤 3：安装 PyTorch（带 CUDA）

pip install torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124
💡

DGX Spark 的 Grace Hopper 架构建议使用 CUDA 12.4+ 和 PyTorch 2.6+ 以获得最佳性能。也可以通过 nvidia-smi 查看已安装的 CUDA 版本来选择匹配的 PyTorch 版本。

步骤 4：安装 FlashAttention-2（强烈推荐）

pip install flash-attn --no-build-isolation
⚡

FlashAttention-2 可以将训练速度提升 2-3 倍，同时减少显存占用。DGX Spark 的 Grace Blackwell 架构完全兼容。

步骤 5：安装 LLaMA-Factory

# 克隆仓库
git clone https://github.com/hiyouga/LLaMA-Factory.git
cd LLaMA-Factory

# 安装依赖（包含所有特性）
pip install -e ".[torch,metrics]"
💡

安装完成后，可以运行 llamafactory-cli version 验证安装是否成功。

步骤 6：配置项目目录

将本教程附带的 qwen-finetune/ 目录复制到你的工作目录，或者直接在 LLaMA-Factory/ 目录内创建软链接：

# 方式一：直接复制
cp -r ~/Desktop/qwen-finetune ~/projects/

# 方式二：符号链接（推荐，方便修改）
ln -s ~/Desktop/qwen-finetune ~/projects/qwen-finetune

5 数据集准备与格式说明

📝 Alpaca 格式

本教程使用的数据集采用 Alpaca 格式，这是 LLaMA-Factory 支持的标准格式之一。每条数据包含三个字段：

字段含义本实验值
instruction用户指令/问题如"什么是 KYC？"
input可选的额外输入（本实验为空）""
output期望的模型回复专业金融解答

📊 数据来源：BELLE 中文指令数据集

本教程的数据集来自 BELLE (Be Everyone's Large Language model Engine) 项目，具体使用 BelleGroup/train_0.5M_CN 子集。

  • 原始数据集包含 519,000 条中文指令数据，涵盖各类任务
  • 从中通过金融关键词（银行、贷款、利率、投资、保险、KYC、Basel 等）筛选出 63 条高质量金融问答
  • 关键词涵盖银行、投资、宏观经济、公司金融、区块链等子领域
📌

提取脚本 在项目 scripts/extract_belle_finance_v2.py 和 scripts/filter_belle_finance.py 中，如需更多数据可以调整阈值重新运行。

📄 数据样例

{
  "instruction": "解释以下金融术语的含义。\n基金、股票、投资回报率",
  "output": "基金是由多个投资者合资形成的一种集合型投资工具..."
}

🔗 数据集注册

data/dataset_info.json 文件告诉 LLaMA-Factory 如何加载我们的自定义数据集：

{
  "financial_qa": {
    "file_name": "financial_qa.json",
    "formatting": "alpaca",
    "columns": {
      "prompt": "instruction",
      "query": "",
      "response": "output"
    }
  }
}
⚠️

使用 LLaMA Board Web UI 时，需要在 "数据集"选项卡 中将 dataset_dir 设置为你的数据目录路径，并在训练配置中引用 dataset_info.json 中定义的名称（如 financial_qa）。

统计项数值
来源BelleGroup/train_0.5M_CN (HuggingFace)
原始数据集大小519,000 条
筛选后样本数63 条
筛选方式金融关键词匹配（银行、投资、保险、利率等）

📌

生产建议：BELLE 数据集质量较高，63 条数据已可看到明显的微调效果。如需更多数据，可以使用项目中 scripts/extract_belle_finance_v2.py 和 scripts/filter_belle_finance.py 脚本，调整关键词列表从 train_0.5M_CN（51.9 万条）或更大的 train_2M_CN（200 万条）中提取更多金融数据。

🤔 为什么只用 63 条而不是整个 51.9 万条数据集训练？

🎯 任务目标决定数据选择

我们的目标是让模型在金融领域回答得更专业，而不是让它学会通用的指令跟随能力。BELLE train_0.5M_CN 是全领域数据集——涵盖翻译、写作、编程、数学、创意、角色扮演、情感分析等。如果直接用全部 51.9 万条训练：

训练方式效果
全 51.9 万条（泛领域）模型学了各类通用能力，但金融知识提升微乎其微（金融数据占比 < 1%）
只取金融相关 63 条模型在金融问题上显著提升，原有的通用能力不受影响

⏱️ 训练效率

按我们的配置（batch=2, gradient accumulation=4, 5 epochs）：

数据量训练步数预计时间（8GB GPU）
63 条~315 步~10 分钟 ✅
51.9 万条~260 万步数天 ❌

作为教程和实验，10 分钟跑完完整流程的价值远高于跑几天的黑盒训练。

🧪 可观测性

用 63 条数据训练，你可以清晰观察到：

  • loss 从 ~2.5 快速下降到 ~0.1
  • 微调前后回答质量的明显差异
  • 每个 checkpoint 之间的变化

用全量数据训练几天，loss 缓慢下降，你几乎无法直观判断"训练是否真的有效"。

🔍 提取深度

实际上我们只扫描了前 6,000 条（约 1%）就提取到了 63 条金融相关。如果扫描全部 51.9 万条，预计可以提取到 几千条 金融数据，效果会更好。

💡 如果你想用更多数据

修改 scripts/extract_belle_finance_v2.py 中的两条阈值：

参数当前值建议值预期结果
max_scan只看前 5 万条改为扫描全部提取 500-1000 条
max_count100 条改为 500更丰富的金融覆盖

这样在可接受的训练时间（30-60 分钟）内，用更大规模的金融数据集微调，效果会明显更好。

打个比方：你让一个学生精读一本百科全书，他什么都懂一点但都不深；你让他精读一本金融专业书，金融考试就一定拿高分。63 条金融专用数据正是那本"金融专业书"。

📊 训练参数详解

核心训练配置 config/train_qwen_qlora.yaml 中的关键参数说明：

参数值说明
finetuning_type: loralora使用 LoRA 微调方法
quantization_bit: 444-bit 量化 → QLoRA
lora_target: allall对模型所有线性层应用 LoRA，效果最好
lora_rank: 88LoRA 秩。8 在效果和效率间取得平衡，越大越强但也越贵
per_device_train_batch_size: 22每个 GPU 的 batch size。DGX Spark 可以设更高
gradient_accumulation_steps: 44梯度累积 4 步，等效 batch size = 2×4 = 8
learning_rate: 2.0e-42e-4LoRA 常用学习率，比全参数微调高 10 倍
num_train_epochs: 55训练 5 轮，小数据集建议多训练几轮
bf16: truetrue使用 bfloat16 精度，DGX Spark 原生支持
flash_attn: autoauto自动启用 FlashAttention 加速

📈 训练进度解读

训练过程中你会看到类似下面的日志输出：

  0%|...| 1/5 [00:32<02:38, 39.6s/it]
  {'loss': 2.3841, 'grad_norm': 0.8921, 'lr': 5.2e-05, 'epoch': 0.1}

  20%|...| 1/5 [00:32<02:38, 39.6s/it]
  {'loss': 1.8452, 'grad_norm': 0.6541, 'lr': 8.6e-05, 'epoch': 0.2}

  40%|...| 2/5 [01:04<01:36, 32.1s/it]
  {'loss': 1.2103, 'grad_norm': 0.4452, 'lr': 1.2e-04, 'epoch': 0.4}
  ...

关键指标解读：

  • loss：交叉熵损失，越低越好。从 2.5 左右开始，降至 0.5 以下说明模型已较好拟合
  • grad_norm：梯度范数，稳定在 1.0 附近为佳，过大说明训练不稳定
  • lr：当前学习率，随 cosine 调度器变化
  • epoch：当前训练轮次（小数表示部分轮次）

观察项正常范围异常信号解决办法

🔧 DGX Spark 特有优化

🚀

DGX Spark 的 Grace Blackwell 架构优化建议：

  • 由于 128GB 统一内存，可以安全地将 per_device_train_batch_size 提高到 4-8
  • 启用 pure_bf16: true 利用 Grace CPU 的原生 bf16 支持
  • dataloader_num_workers 可设为 8-16（充分利用 DGX Spark 的 20 核 ARM CPU）
  • 建议在训练时关闭 GUI 应用，将更多资源留给 GPU

9 模型评估与对话推理

💬 方式一：在 Web UI 中直接对话

问：什么是区块链技术以及它的主要应用场景？
问：请给我推荐一只近期经济增长潜力较高的股票
问：给出一份个人理财建议，月收入10000元，月开销7000元
问：什么是金融风险管理？它对银行业的重要性是什么？

11 常见问题与故障排除

🔴

CUDA Out of Memory (OOM)

现象：训练开始时显示 CUDA out of memory 错误。

解决：

  1. 减小 per_device_train_batch_size 到 1
  2. 减小 cutoff_len 到 1024
  3. 确认 quantization_bit: 4 已设置
  4. 关闭其他 GPU 应用（浏览器、PyCharm 等）
  5. 重启终端释放 GPU 缓存
🔴

bitsandbytes 安装失败

现象：安装 bitsandbytes 时报错。

解决：

  1. 确认 CUDA 版本匹配：pip install bitsandbytes --index-url https://download.pytorch.org/whl/cu124
  2. DGX Spark ARM64 可用：pip install https://github.com/jllllll/bitsandbytes-windows-webui/releases/download/wheels/bitsandbytes-0.43.3-cp311-cp311-win_amd64.whl（替换为 ARM 版本）
  3. 或使用 pip install -U bitsandbytes 尝试最新版
🔴

Dataset 加载失败

现象：提示 Dataset financial_qa not found。

解决：

  1. 确认 data/dataset_info.json 文件存在于正确的路径
  2. 确认 dataset_dir 指向 data/ 目录（包含 dataset_info.json 的目录）
  3. 确认 dataset_info.json 中的 file_name 路径正确
  4. JSON 文件格式正确，无拼写错误
🔴

FlashAttention 不可用

现象：训练速度很慢，或提示 FlashAttention 相关错误。

解决：

  1. 确认 CUDA 版本 ≥ 11.6：nvcc --version
  2. DGX Spark 的 Blackwell 架构完全兼容 FlashAttention-2
  3. 如果编译失败，设置 flash_attn: false 回退到标准 attention
🔴

模型下载太慢

现象：从 HuggingFace 下载 Qwen2.5-7B 耗时过长。

解决：

  1. 设置镜像站加速：export HF_ENDPOINT=https://hf-mirror.com
  2. 先手动下载：huggingface-cli download Qwen/Qwen2.5-7B-Instruct --local-dir ./models/qwen2.5-7b
  3. 然后在配置中将 model_name_or_path 指向本地路径

12 附录：参数详解

📖 关键训练参数详解

finetuning_type — 微调类型

lora: LoRA 微调（推荐）
full: 全参数微调（需要大量显存）
freeze: 冻结部分层微调
本实验：lora

quantization_bit — 量化位数

4: 4-bit 量化（QLoRA，约 6GB 显存）
8: 8-bit 量化（约 10GB 显存）
none: 不量化（约 16GB 显存）
本实验：4（最大化降低显存需求）

lora_rank — LoRA 秩

LoRA 矩阵的秩（rank），影响微调的"容量"：
8: 推荐值，效果与效率的平衡点
16: 更强表达力，适合复杂任务
32: 最强但训练更慢
64+: 收益递减，通常不需要
本实验：8

lora_alpha — LoRA 缩放因子

控制 LoRA 更新的缩放比例。
通常设为 lora_rank × 2（如 rank=8 则 alpha=16）。
值越大，LoRA 更新的影响越大。
本实验：16

learning_rate — 学习率

LoRA/QLoRA 通常比全参数微调高 10 倍：
1e-4: 激进，适合大数据集
2e-4: 通常推荐值 ✅
5e-4: 快速但可能不稳定
1e-5: 保守，适合极小数据集
本实验：2e-4

per_device_train_batch_size — 每设备 batch size

每个 GPU 每次前向传播的样本数。
显存充足时可提高；出现 OOM 时减小。
DGX Spark (128GB 统一内存) 可设到 4-8。
本实验：2（保守设置，运行稳定）

gradient_accumulation_steps — 梯度累积步数

累积 N 步的梯度后再更新参数。等效模拟更大的 batch size。
总 batch size = per_device_train_batch_size × gradient_accumulation_steps × GPU数量
本实验：4（等效 batch = 2×4 = 8）

num_train_epochs — 训练轮数

整个训练集被遍历的次数。
小数据集（百条级）需要 5-20 轮
大数据集（万条级）通常 2-5 轮即够
本实验：5（仅 10 条数据，5 轮已足够）

lr_scheduler_type — 学习率调度器

cosine: 余弦退火，平滑下降（推荐）
linear: 线性下降
constant: 恒定学习率
polynomial: 多项式衰减
本实验：cosine

cutoff_len — 最大序列长度

模型能处理的最大 token 数。
越长，占用显存越多，但能处理更长的上下文。
512: 节省显存，适合简单问答
1024: 适中
2048: 推荐，大多数场景足够 ✅
4096+: 需大量显存
本实验：2048

🧪 进阶参数调优建议

你的显存推荐配置预期效果
4 GBQLoRA + batch=1 + cutoff=512 + rank=4勉强运行，速度较慢
8 GB ✅本实验配置: QLoRA + batch=2 + cutoff=2048 + rank=8流畅运行，效果优良
16 GBQLoRA + batch=4 + cutoff=4096 + rank=16大 batch，长上下文
24 GB+LoRA (8-bit) + batch=8 + cutoff=4096更大的训练容量
128 GB (DGX Spark)QLoRA + batch=8 + cutoff=4096 + rank=16极速训练，充分利用统一内存
